# 图文卡风格提炼与保存

用于从参考图片、已有图文卡、截图、品牌视觉或文字描述中提炼可复用的小红书图文卡风格，并保存为 `styles/{style-name}.md`。

## 提炼目标

不要只描述参考图表面元素。要提炼可复用的视觉 DNA：

- 风格定位：适合什么内容，不适合什么内容。
- 视觉关键词：3-8 个稳定关键词。
- 色彩系统：背景色、主色、辅色、文字色、强调色、线条/容器色。
- 字体与排版气质：标题、正文、数字、标签分别是什么感觉。
- 版式网格：标题区、主体区、底部区、边距、留白、阅读顺序。
- 组件系统：卡片、边框、分割线、箭头、图标、纹理、装饰元素。
- 信息密度：每页适合几个模块、每个模块几行字。
- 页型适配：封面页、流程页、时间线页、关系页、对比页、清单页、框架页、金句页分别怎么做。
- 推荐提示词骨架：使用 MoreImg 的字段化提示词结构。
- 负面约束：最不能混入的风格、颜色、材质、布局和错误文字密度。
- 质量检查：生成后如何判断这个风格有没有跑偏。

## 访谈问题

只在必要时提问；已有信息足够时直接提炼。

1. 这个风格主要用于什么内容？
2. 默认比例是否为小红书 `3:4`、`4:5`，还是其它比例？
3. 文字密度偏好是极简、中等信息量，还是信息图密度？
4. 是否要保存为可复用风格？如果保存，风格名和别名是什么？
5. 是否有不能出现的元素：日期、logo、水印、人物、品牌名、深色背景等？
6. 是否允许和已有风格混合？默认不混合。

## 保存格式

风格文件至少包含：

```markdown
# 风格名称

别名：

## 风格定位

## 视觉 DNA

## Style Lock

## 色彩建议

## 页型适配

## 图片构图规则

## 默认输出参数

## 排版规则

## 信息密度

## 推荐提示词骨架

## 负面约束

## 质量检查
```

## 调用说明

保存后告诉用户可以这样调用：

```text
用 MoreImg 调用 {风格名}，把这篇文章做成小红书图文卡提示词。
```

## 边界

- 不要把风格写进 `SKILL.md`。
- 不要把一次性内容、标题、具体文案保存到风格文件。
- 不要保存参考图里的 logo、水印、错字、错误构图或偶然瑕疵。
- 不要混入历史任务的风格，除非用户明确要求 hybrid。
