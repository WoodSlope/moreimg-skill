# 图文卡提示词返修

用于用户对 MoreImg 生成的 `cards.md` 或 `prompts.md` 提出局部修改时。

## 修改原则

- 默认只修改用户点名的页面或字段，不重做整套。
- 保持原 Style Lock、页型、参数标签、发布配文不变，除非用户明确要求修改。
- 如果问题是概念层级错误，优先修改 `cards.md` 的语义映射，再同步修改 `prompts.md` 的视觉锚点和避免项。
- 如果问题只是画面风格、配色、排版或视觉锚点，通常只修改 `prompts.md`。
- 如果问题是入图文字不准，先修改 `cards.md` 的入图文字，再同步更新 `prompts.md` 的内容层级。

## 常见修改类型

### 1. 主视觉偏离文案

处理：

- 重新判断主概念、主关系、辅助概念、旁注概念。
- 如果旁注概念被画进主视觉，先判断它是否应当 `不入图`；若应不入图，把它从 `内容层级` 和 `视觉锚点` 中移除，只在 `避免` 中作为禁用对象出现。
- 重写该页 `视觉锚点`。
- 在 `避免` 中明确禁止错误视觉隐喻。

### 2. 文字太多

处理：

- 压缩 `cards.md` 的入图文字。
- 保留标题、3-5 个短信息条、1 句总结。
- 同步修改 `prompts.md` 的内容层级。

### 3. 风格跑偏

处理：

- 检查是否使用了正确 Style Lock。
- 加强 `画面气质`、`色彩`、`避免`。
- 不要为了修一页而换整套风格。

### 4. 只改发布配文

处理：

- 只修改 `cards.md` 的 `发布配文`。
- 不修改 prompts。

## 返修输出格式

用户只要求修改一页时，输出该页的新内容即可：

```markdown
## prompts.md 修改

### 内容页2/2
[3:4, 1K] ...
```

如果同时影响 `cards.md` 和 `prompts.md`，输出两个对应片段。

## 需要追问的情况

- 用户只说“不好看”，没有指出问题。
- 用户没有说要改第几页。
- 用户要求“大改风格”，可能影响整套一致性。

## 不需要追问的情况

- 用户明确指出页码和问题。
- 用户提供了新文字。
- 用户指出概念层级错误。
- 用户要求按原风格重写某一页提示词。
