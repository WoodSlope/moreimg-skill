# 文生图提示词

## 使用风格

xhs-tech-knowledge｜小红书科技知识卡

## Style Lock

银灰蓝科技知识卡，浅灰蓝或银灰渐变背景，细网格纸纹，线框透视图标，几何无衬线黑体，标题清晰大字，模块化信息区，留白充足，低到中等文字密度，手机端可读。中文优先，保留 Workflow、Skill、Tool、API、Agent、SaaS 等必要英文术语。统一使用细线框、轻描边模块、节点、箭头、流程轨道、状态标签、关系线。不要混入深色科技大屏、紫色霓虹渐变、奶油风、黑金风、卡通插画或复杂 3D。

## 提示词

### 封面页

[3:4, 1K] 生成一张小红书图文卡片，页型是「封面页」，主题是「Workflow 概念全溯源」。
画面气质：银灰蓝科技知识卡，浅灰蓝背景，细网格纸纹，线框透视图标，几何无衬线黑体，留白充足。
排版：上方强标题区，中部一条贯穿多个时代节点的流程轨道，底部短标语，边距稳定，手机端可读。
内容层级：大标题「Workflow 概念全溯源」，副标题「从传统软件到 AI 智能体」，提示「讲清 Workflow、Skill、微服务」，总结「Workflow 是流程编排思想，不是 AI 才有的新概念」。
视觉锚点：线框流程轨道从传统软件节点延伸到 AI 智能体节点，表达 Workflow 作为长期存在的流程编排主线。
色彩：浅灰蓝和银灰背景，深海蓝文字与线条，少量青蓝强调节点，低饱和，高对比。
避免：大段正文、复杂箭头、随机英文假字、深色科技大屏、把 Workflow 画成单一 AI 软件功能。

### 内容页1/2

[3:4, 1K] 生成一张小红书图文卡片，页型是「时间线页」，主题是「Workflow 的发展脉络」。
画面气质：银灰蓝科技知识卡，细网格纸纹，线框图标，几何无衬线黑体，模块化信息区，留白充足。
排版：上方标题区，中部四阶段纵向时间线，每个阶段一个线框节点图标和一条短信息，底部一句总结。
内容层级：大标题「Workflow 的发展脉络」，要点包括「传统软件：流程写在代码里」「自动化工具：流程拖拽可视化」「云端平台：流程在线运行」「AI 智能体：执行骨架」，总结「被调度的能力一直在变」。
视觉锚点：四阶段纵向时间线，节点依次为代码模块、拖拽节点、云端 API、AI Agent 调度；不要把微服务放进时间线或画成云端架构。
色彩：浅灰蓝背景，深海蓝文字与线条，青蓝强调关键节点，弱灰蓝边框，背景纹理轻。
避免：把微服务画成独立发展阶段、云端服务模块、服务器集群、复杂系统架构图、箭头过多、密集小字、不可读中文。

### 内容页2/2

[3:4, 1K] 生成一张小红书图文卡片，页型是「关系页」，主题是「Workflow 和 Skill 的核心关系」。
画面气质：银灰蓝科技知识卡，浅灰蓝或银灰渐变背景，细网格纸纹，线框透视图标，几何无衬线黑体，模块化信息区，留白充足。
排版：上方纠错区写误解与正确理解，中部是主关系图，Workflow 作为流程骨架主轴向多个 Skill 能力积木发出简洁调度箭头，底部金句强调；画面中不要放微服务区域。
内容层级：大标题「一页讲清核心关系」，副标题/提示「误解：Workflow 只是画线连功能；正确：Workflow 是通用流程编排思路」，要点包括「Workflow：管调度」「Skill：管能力」，总结「Workflow 是流程骨架，Skill 是能力积木」。
视觉锚点：主视觉只表达 Workflow 调度 Skill：一条清晰流程骨架连接多个能力积木；微服务是原文中的部署方式联想，不进入视觉锚点，不画任何云端、服务器、微服务模块或连接箭头。
色彩：浅灰蓝背景，深海蓝文字与主线条，青蓝强调 Workflow 主轴，Skill 积木使用白色轻描边模块。
避免：把 Workflow、Skill、微服务画成并列三层架构；不要做三栏平权关系图；不要出现云端服务模块、服务器、云朵、微服务图标；不要让微服务和 Workflow / Skill 发生任何视觉连接；不要概念堆叠、箭头过多、低对比文字。
