# 文生图提示词

## 使用风格

xhs-tech-knowledge｜小红书科技知识卡

## Style Lock

银灰蓝科技知识卡，浅灰蓝或银灰渐变背景，细网格纸纹，线框透视图标，几何无衬线黑体，标题清晰大字，模块化信息区，留白充足，低到中等文字密度，手机端可读。中文优先，保留 AI、RAG、API、Workflow 等必要英文术语。统一使用细线框、轻描边模块、节点、箭头、流程轨道、状态标签、关系线。不要混入深色科技大屏、紫色霓虹渐变、奶油风、黑金风、卡通插画或复杂 3D。

## 提示词

### 封面页

[3:4, 1K] 生成一张小红书图文卡片，页型是「封面页」，主题是「AI 知识库建设顺序」。
画面气质：银灰蓝科技知识卡，浅灰蓝背景，细网格纸纹，线框透视图标，几何无衬线黑体，留白充足。
排版：上方强标题区，中部一个知识资产线框主视觉，底部短标语，边距稳定，手机端可读。
内容层级：大标题「AI 知识库别急着上 RAG」，副标题「先理清内容源、切分方式、反馈闭环」，提示「企业知识库接入 AI 前，真正要先准备什么」，总结「知识库不是文件夹，RAG 也不是第一步」。
视觉锚点：知识资产主线从内容源整理延伸到 AI 调用，表达先治理内容再谈技术调用；RAG 只作为后段小标签。
色彩：浅灰蓝和银灰背景，深海蓝文字与线条，少量青蓝强调关键节点，低饱和，高对比。
避免：把 RAG 画成唯一主角、复杂系统架构图、过多小字、随机英文假字、深色科技大屏。

### 内容页1/2

[3:4, 1K] 生成一张小红书图文卡片，页型是「流程页」，主题是「知识库建设的三层准备」。
画面气质：银灰蓝科技知识卡，细网格纸纹，线框图标，几何无衬线黑体，模块化信息区，留白充足。
排版：上方标题区，中部三段纵向流程节点，底部一句总结，节点间距稳定，手机端可读。
内容层级：大标题「知识库建设的三层准备」，要点包括「1 内容源：确认哪些资料能用」「2 结构化：决定怎么切分」「3 反馈闭环：让错误能被修正」，总结「不要把混乱搬进系统」。
视觉锚点：三段递进流程图，内容源治理在前，结构化切分居中，反馈闭环在后，表现建设顺序而非并列清单。
色彩：浅灰蓝背景，深海蓝文字与线条，青蓝强调流程主线，白色轻描边模块，背景纹理轻。
避免：把三层画成无顺序并列清单、把 RAG 或向量库作为主步骤、箭头过多、密集小字、不可读中文。

### 内容页2/2

[3:4, 1K] 生成一张小红书图文卡片，页型是「关系页」，主题是「RAG 与知识库建设的关系」。
画面气质：银灰蓝科技知识卡，浅灰蓝或银灰渐变背景，细网格纸纹，线框透视图标，几何无衬线黑体，模块化信息区，留白充足。
排版：上方纠错区写误解与正确理解，中部是前置基础到技术调用的关系图，底部金句强调。
内容层级：大标题「RAG 只是最后一步」，副标题/提示「误解：接入 RAG = 有了企业知识库；正确：RAG 之前还有治理和结构」，要点包括「内容治理：决定什么能被用」「结构设计：决定怎么被检索」「RAG：决定怎么被调用」，总结「先把内容变成可靠资产」。
视觉锚点：内容治理和结构设计作为两个前置基础模块，RAG 位于后段调用节点，表达 RAG 依赖前置内容资产，不是知识库建设的中心。
色彩：浅灰蓝背景，深海蓝文字与主线条，青蓝强调前置基础，RAG 节点用较低权重处理。
避免：把 RAG 放在画面中心支配全部、暗示 RAG 替代内容治理、复杂三层架构、概念堆叠、低对比文字。
