# 图文笔记内容

## 页面结构

### 封面页
- 页型：封面页
- 页面目标：建立排错原则：Agent 失败时，不要先重写 prompt，而要先定位失败位置。
- 入图文字：
  - 大标题：Agent 失败别急着改 Prompt
  - 副标题：先把失败拆开看
  - 标签：输入 / 计划 / 工具 / 中间产物 / 微调
  - 标语：可靠排错，不靠一次猜对
- 语义映射：用终端警告窗口和五个状态节点表达“先定位，再修改”。
- 避免误读：不要变成纯代码截图或密集报错日志。

### 内容页1/3
- 页型：对比页
- 页面目标：纠正常见误区：失败后马上重写提示词或换模型。
- 入图文字：
  - 大标题：先别重写提示词
  - 误区：失败 = Prompt 不行
  - 正确：先定位失败在哪一步
  - 关键：输入、计划、工具、中间产物
  - 总结：Prompt 是最后才微调的变量
- 语义映射：左右对照，左侧是错误反应，右侧是稳定排错思路。
- 避免误读：不要暗示 prompt 不重要；重点是修改顺序和定位方法。

### 内容页2/3
- 页型：流程页
- 页面目标：把五步排错链路压缩成可记住的执行流程。
- 入图文字：
  - 大标题：五步排错链路
  - 1 输入检查
  - 2 计划检查
  - 3 工具检查
  - 4 中间产物检查
  - 5 提示词微调
  - 总结：先观察，再修正
- 语义映射：五个发光状态节点按顺序连接，表达排错路径。
- 避免误读：不要把五步画成并列清单，要表现顺序和递进。

### 内容页3/3
- 页型：清单页
- 页面目标：提供每一步该看什么的短检查项。
- 入图文字：
  - 大标题：每一步看什么？
  - 输入：目标、路径、上下文
  - 计划：步骤是否拆对
  - 工具：权限、参数、服务状态
  - 产物：日志、草稿、中间文件
  - 微调：一次只改一个变量
  - 总结：让每一步都可观察
- 语义映射：终端检查面板，每个模块一个状态点和短标签。
- 避免误读：不要扩写成长教程；保持短句和状态检查感。

## 发布配文

Agent 任务失败时，不要一上来就重写提示词，更不要马上换模型。

更稳定的方式是先定位：输入是否完整，计划是否拆对，工具调用有没有权限或参数问题，中间产物有没有暴露错误。

最后再改 prompt，而且最好一次只改一个变量。

记住这条链路：输入检查 → 计划检查 → 工具检查 → 中间产物检查 → 提示词微调。
