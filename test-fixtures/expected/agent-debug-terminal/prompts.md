# 文生图提示词

## 使用风格

xhs-terminal-tech-magazine｜小红书终端科技杂志风

## Style Lock

深色终端科技杂志风，深黑或深墨绿背景，细网格暗纹，青绿色霓虹高光，玻璃拟态暗面板，细线 HUD 元素，现代无衬线黑体，标题大而清楚，模块化信息区，手机端可读。中文优先，保留 Agent、Prompt、Tool、API、LLM 等必要英文术语。统一使用细线 HUD 框、半透明暗面板、状态圆点、发光胶囊、终端窗口、流程节点。不要混入银灰蓝浅色知识卡、法式电商、机构报告黑线网格、奶油风、黑金奢华、卡通插画或复杂 3D。

## 提示词

### 封面页

[3:4, 1K] 生成一张小红书图文卡片，页型是「封面页」，主题是「Agent 排错流程」。
画面气质：深色终端科技杂志风，深黑或深墨绿背景，细网格暗纹，青绿色高光，玻璃拟态暗面板，细线 HUD 元素，现代无衬线黑体。
排版：上方强标题区，中部一个终端警告窗口连接五个状态节点，底部状态标语，边距稳定，手机端可读。
内容层级：大标题「Agent 失败别急着改 Prompt」，副标题「先把失败拆开看」，要点包括「输入」「计划」「工具」「中间产物」「微调」，总结「可靠排错，不靠一次猜对」。
视觉锚点：终端警告窗口和五个发光状态节点，表达先定位失败位置再修改 Prompt。
色彩：深黑/深墨绿背景，白色正文，青绿色和冰蓝高光，少量黄色警示点，柔和发光，高对比。
避免：纯代码截图堆叠、密集报错日志、赛博朋克游戏 UI、随机英文假字、不可读中文。

### 内容页1/3

[3:4, 1K] 生成一张小红书图文卡片，页型是「对比页」，主题是「Agent 失败后的第一反应」。
画面气质：深色终端科技杂志风，深色背景，青绿色高光，暗色玻璃面板，终端状态标签，留白充足。
排版：上方标题区，中部左右双栏暗面板，左侧红黄警示状态表示误区，右侧青绿色状态表示正确路径，底部一句总结。
内容层级：大标题「先别重写提示词」，副标题/提示「失败不等于 Prompt 一定不行」，要点包括「误区：马上换模型」「正确：先定位步骤」「关键：输入 / 计划 / 工具 / 产物」，总结「Prompt 是最后才微调的变量」。
视觉锚点：左右对照面板，左侧是闪烁错误提示，右侧是稳定排错状态链路。
色彩：深黑背景，白色文字，青绿色正确状态，少量黄色/红色警示，暗面板边框统一。
避免：暗示 Prompt 不重要、文字过多、警示色过量、复杂背景抢信息。

### 内容页2/3

[3:4, 1K] 生成一张小红书图文卡片，页型是「流程页」，主题是「Agent 五步排错链路」。
画面气质：深色终端科技杂志风，细网格暗纹，发光状态节点，HUD 连线，模块化信息区，手机端可读。
排版：上方标题区，中部五个纵向发光状态节点按顺序连接，每个节点一个短标签，底部状态总结条。
内容层级：大标题「五步排错链路」，要点包括「1 输入检查」「2 计划检查」「3 工具检查」「4 中间产物检查」「5 提示词微调」，总结「先观察，再修正」。
视觉锚点：五个顺序状态节点组成排错链路，从输入到 prompt 微调逐步递进。
色彩：深墨绿背景，白色文字，青绿色主连线，冰蓝节点光，少量黄色强调当前检查点。
避免：把五步画成无顺序并列清单、过多小字、密集数据大屏、不可读中文。

### 内容页3/3

[3:4, 1K] 生成一张小红书图文卡片，页型是「清单页」，主题是「Agent 排错检查项」。
画面气质：深色终端科技杂志风，终端检查面板，青绿色状态点，细线 HUD 框，现代无衬线黑体。
排版：上方标题区，中部五个终端检查模块纵向排列，每个模块一个状态点和短说明，底部一句总结。
内容层级：大标题「每一步看什么？」，要点包括「输入：目标、路径、上下文」「计划：步骤是否拆对」「工具：权限、参数、服务状态」「产物：日志、草稿、中间文件」「微调：一次只改一个变量」，总结「让每一步都可观察」。
视觉锚点：终端检查列表，每项显示通过/待查状态，强调可观察、可定位、可修正。
色彩：深黑背景，白色正文，青绿色状态点，灰蓝辅助文字，统一暗面板边框。
避免：长教程文字、纯代码截图、过多状态灯、随机英文假字、混入浅色知识卡风格。
