# 文生图提示词

## 使用风格

xhs-tech-knowledge｜小红书科技知识卡

## Style Lock

银灰蓝科技知识卡，浅灰蓝或银灰渐变背景，细网格纸纹，线框透视图标，几何无衬线黑体，标题清晰大字，模块化信息区，留白充足，低到中等文字密度，手机端可读。中文优先，保留 AI、RAG、API 等必要英文术语。统一使用细线框、轻描边模块、节点、箭头、流程轨道、状态标签、关系线。不要混入深色科技大屏、紫色霓虹渐变、奶油风、黑金风、卡通插画或复杂 3D。

## 提示词

### 封面页

[3:4, 1K] 生成一张小红书图文卡片，页型是「封面页」，主题是「AI 知识库内容源治理」。
画面气质：银灰蓝科技知识卡，浅灰蓝背景，细网格纸纹，线框透视图标，几何无衬线黑体，留白充足。
排版：上方强标题区，中部文件散落到知识资产的转换主视觉，底部短标语，边距稳定，手机端可读。
内容层级：大标题「AI 知识库先别急着接模型」，副标题「内容源混乱，RAG 只会放大问题」，要点包括「内容源」「切分」「反馈闭环」，总结「先把企业内容变成可靠资产」。
视觉锚点：散落文件、部门资料和维护标签汇聚成一个干净的知识资产模块，表达先治理内容再接入 AI。
色彩：浅灰蓝和银灰背景，深海蓝文字与线条，少量青蓝强调治理主线，低饱和，高对比。
避免：模型选型对比、复杂架构图、把 RAG 画成唯一主角、过多小字、随机英文假字。

### 内容页1/3

[3:4, 1K] 生成一张小红书图文卡片，页型是「对比页」，主题是「AI 知识库建设的顺序误区」。
画面气质：银灰蓝科技知识卡，细网格纸纹，线框图标，几何无衬线黑体，模块化信息区，留白充足。
排版：上方标题区，中部左右对照面板，左侧表示技术组件先行，右侧表示内容源治理先行，底部一句总结。
内容层级：大标题「问题不在模型，先看内容源」，副标题/提示「常见误区：先选模型、向量库、RAG 框架」，要点包括「正式来源」「维护责任」「过期 / 冲突 / 越权 / 无人维护」，总结「别把混乱搬进系统」。
视觉锚点：左右对照图，左侧是模型和向量库标签堆叠，右侧是资料来源、版本、权限、责任人四个治理模块。
色彩：浅灰蓝背景，深海蓝文字与线条，青蓝强调正确路径，少量灰蓝表示风险项。
避免：暗示模型能力不重要、把技术组件画成主要解决方案、箭头过多、密集小字、不可读中文。

### 内容页2/3

[3:4, 1K] 生成一张小红书图文卡片，页型是「流程页」，主题是「AI 知识库建设顺序」。
画面气质：银灰蓝科技知识卡，细网格纸纹，线框图标，几何黑体，模块化信息区，手机端可读。
排版：上方标题区，中部四步纵向流程轨道，底部一句总结，模块间距稳定，留白充足。
内容层级：大标题「更稳的建设顺序」，要点包括「1 内容源治理」「2 结构化切分」「3 检索与 RAG」「4 反馈闭环」，总结「让知识持续变好」。
视觉锚点：四步纵向流程图，从内容源治理到结构化切分，再到检索与 RAG，最后回到反馈闭环，表达持续改进。
色彩：浅灰蓝背景，深海蓝文字与主线条，青蓝强调流程主轴，白色轻描边模块。
避免：把四步画成无顺序并列清单、复杂系统架构、过多图标、低对比文字。

### 内容页3/3

[3:4, 1K] 生成一张小红书图文卡片，页型是「清单页」，主题是「AI 知识库接入前检查项」。
画面气质：银灰蓝科技知识卡，浅灰蓝背景，细网格纸纹，线框检查面板，几何无衬线黑体，留白充足。
排版：上方标题区，中部四个检查模块纵向排列，每个模块一个短问题和状态标签，底部总结条。
内容层级：大标题「接入 AI 前先问 4 个问题」，要点包括「哪些资料是正式来源？」「哪些内容已经过期？」「谁负责维护和修正？」「错误答案如何反馈？」，总结「知识库要可维护、可检索、可改进」。
视觉锚点：上线前检查面板，四个问题像状态检查项一样排列，强调可维护、可检索、可改进。
色彩：浅灰蓝背景，深海蓝文字，青蓝状态点，弱灰蓝边框，低饱和，高对比。
避免：长教程文字、密集表格、随机装饰线、不可读中文、混入深色终端风格。
