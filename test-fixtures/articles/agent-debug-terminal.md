# AI Agent 排错流程：不要一上来就重写

## 文章目标

把 Agent 执行失败后的排错流程讲清楚，适合做深色终端科技风小红书知识卡。

## 正文

很多人用 AI Agent 做自动化任务时，遇到失败的第一反应是重写提示词，甚至直接换模型。但真正稳定的排错方式，应该先把失败拆开看。

第一步是看输入。任务目标是否清楚？文件路径是否存在？上下文是否缺失？如果输入本身不完整，后面再强的模型也只能猜。

第二步是看计划。Agent 是否把任务拆成了正确步骤？有没有把“查询资料”“处理文件”“生成结果”混在一起？计划错了，工具调用通常也会跟着错。

第三步是看工具调用。联网、代码执行、文件读取、图片生成这些工具都有边界。失败不一定是模型不会做，也可能是工具没有权限、参数格式错误、路径写错或外部服务不可用。

第四步是看中间产物。很多 Agent 任务不是一步完成的，中间文件、日志、草稿和结构化结果都能暴露问题。不要只看最终答案。

第五步才是改提示词。改提示词也不要一次改很多变量，最好只改目标、约束、输出格式或工具选择中的一个点，这样才能知道是哪一处产生了效果。

一个简单排错链路可以这样记：输入检查 → 计划检查 → 工具检查 → 中间产物检查 → 提示词微调。

真正可靠的 Agent 工作流，不是让模型一次猜对，而是让每一步都能被观察、被定位、被修正。

## 发布配文草稿

Agent 失败时，不要一上来就重写提示词。先看输入、计划、工具和中间产物，再决定要不要改 prompt。
